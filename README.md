# PHP 언어를 이용한 웹 바이브코딩 가이드
- md 파일 작성시 마지막은 항상 개행문자로 끝남(종료표시)
## 세팅방법
1. ```.gitignore``` 파일과 ```_ukp``` 폴더를 복사 후 개발할 프로젝트에 붙여넣기.
2. ```.gitignore``` 파일 내 폴더 경로를 개발할 프로젝트에 맞게 변경.
3. 추가로 사용할 함수를 개발할 프로젝트의 ```class.ukp.php``` 파일 내에 추가하여 사용.
### 신규 웹개발 하는경우
1. ```www/website``` 폴더 내용 복사, 웹 접근 가능한 폴더에 붙여넣은 후 ```_.php``` 파일 내용 수정.
2. 웹 접근 불가한 곳에 ```cli```, ```prompt``` 폴더 추가.
## 규칙
- 프롬프트 파일은 .md 확장자로 작성
## 폴더구조 설명
- ```cli``` 파일들은 cli 환경에서 실행되고 웹접근 불가.
- ```prompt``` 파일들은 업로드할 md 파일과 소스생성을 위해 작성한 write md 파일, 웹접근 불가.
- ```_css``` 와 ```_js``` 파일들은 ```_view``` 파일들에서 사용.
- ```asset``` 은 정적파일용 폴더, ```upload``` 은 동적파일용 폴더.
- ```_view``` 파일들은 뷰를 위해 사용.
## 파일구조 설명
- ```_.php``` 파일은 각각 웹루트에 위치하고 ```$global```, ```$data``` 변수 설정.
- 웹루트에 접두어가 ```_``` 인 php 파일은 ajax 통신 전용 파일.
- 접두어가 ```_``` 인 ```_view``` 폴더내 파일은 공통뷰를 의미, 아닌경우 페이지별 뷰.
- 파일명에 ```_``` 가 연속해서 두개이상 붙는경우 테스트용 파일이며 커밋되지 않는다.
## 프롬프트 작성 방법
- 기존 프롬프트 참고하여 작성
- 프롬프트 결과가 의도한것과 다르게 나올때 왜 그렇게 나왔는지 물어본 후 프롬프트 수정
- 하나의 명령을 여러줄에 걸쳐서 설명하면 이해도가 떨어짐
- 프롬프트 설명은 ```## 개별파일 상세설명``` 탭에 작성
## 개별파일 상세설명
- 파일 설명은 파일 이름 순서대로 작성
- 파일 이름 옆에 md 파일 수정일 입력
